# 智能陪伴机器人 · 用户端界面设计说明

## 一、设计定位

| 维度 | 说明 |
|------|------|
| 使用人群 | 老年用户为主，家属辅助 |
| 终端形态 | 手机 / 平板 / 智能音箱触屏（移动优先） |
| 设计原则 | 大字号、大按钮、高对比、少层级、语音优先 |
| 与管理后台关系 | 后台配置 → 用户端呈现；后台采集/审核 → 用户端答题/浏览 |

## 二、信息架构（5 Tab + 子页）

```
底部导航
├── 首页     陪伴入口、快捷功能、SOS、采集进度
├── 陪伴     语音/文字双模式对话
├── 采集     预设问卷逐题作答（对应后台「题库管理」）
├── 守护     呼救、用药、家人留言、社区服务
└── 我的     回忆录、档案、设置、亲情账号

子页面（二级，带返回）
├── 我的回忆录 / 家人留言 / 社区服务 / 健康知识
├── 历史对话 / 语音与对话设置
```

## 三、与后台三大模块映射

| 产品模块 | 用户端界面 | 管理后台 |
|---------|-----------|---------|
| 数据采集 | 采集 Tab、首页进度条 | 题库管理、采集任务监控 |
| AI 陪伴 | 首页语音球、陪伴 Tab、设置页 | AI 引擎配置 |
| 养老守护 | 守护 Tab、SOS、留言、社区 | 养老守护管理 |
| 用户档案 | 我的、回忆录、历史对话 | 用户档案管理 |
| 知识库 | 知识问答模式、健康页 | 知识库运营 |

## 四、核心页面说明

### 1. 首页
- 问候语 + 今日用药提醒
- 语音唤醒球（点击 /「小伴小伴」）
- 自由聊天 / 知识问答模式切换
- 8 宫格快捷入口
- **SOS 长按 3 秒呼救**（与管理后台告警联动）
- 信息采集进度条

### 2. 陪伴（对话）
- 气泡式对话，机器人回复支持语音播报标识
- 底部：语音键 + 文字输入 + 发送
- 顶部：自由对话 / 知识库问答切换

### 3. 采集
- 环形进度 + 当前题目卡片
- 语音回答 / 跳过 / 下一题
- 分类标签（童年、家庭、健康等）

### 4. 守护
- SOS 置顶
- 用药打卡列表
- 家人留言摘要
- 社区服务四宫格
- 健康 & 用药知识库入口

### 5. 我的
- 用户头像与陪伴统计
- 功能菜单列表
- 亲情账号在线状态

## 五、适老化规范

- 基础字号 ≥ 17px，标题 ≥ 20px
- 可点击区域最小高度 48px
- 主色 `#2b7de9`，警示 `#ff4d4f`，安全 `#52c41a`
- 关键操作（SOS、发送、保存）使用实心大按钮
- 次要操作（预览、试听）收入「更多」或二级页

## 六、预览方式

```bash
npm install
npm run dev
```

访问 `/portal` 选择「用户端」，或直接打开 `/app/home`。
